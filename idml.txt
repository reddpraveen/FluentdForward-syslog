## Detailed Import Instructions:

### Method 1: Confluence Storage Format (XML) ⭐ RECOMMENDED

**Step-by-step:**

```
1. Open your Confluence page in EDIT mode

2. Click anywhere on the page

3. Open Insert Markup dialog:
   • Windows/Linux: Ctrl + Shift + D
   • Mac: Cmd + Shift + D
   • OR: Click "..." menu → "Insert markup"

4. Open confluence-storage-format.xml in Notepad/TextEdit

5. Select ALL content:
   • Windows/Linux: Ctrl + A
   • Mac: Cmd + A

6. Copy:
   • Windows/Linux: Ctrl + C
   • Mac: Cmd + C

7. Go back to Confluence markup dialog

8. Paste:
   • Windows/Linux: Ctrl + V
   • Mac: Cmd + V

9. Click "Insert" button

10. Save your Confluence page

11. Done! ✅
```

**What you get:**
- ✅ Properly formatted tables
- ✅ Colored panels (blue info, green success, orange warning)
- ✅ Code blocks with syntax highlighting
- ✅ Expandable sections
- ✅ Table of contents
- ✅ All headings and structure

## Troubleshooting:

### ❌ "Insert markup" option not available?

**Solutions:**
1. Check if you have edit permissions
2. Try the HTML method instead
3. Contact Confluence admin - may be restricted
4. Use "Insert → Other macros → Markup"

---

### ❌ XML paste doesn't work?

**Solutions:**
1. Make sure you copied the ENTIRE XML file (from `<?xml` to `</ac:confluence>`)
2. Try pasting in Source Editor mode
3. Use HTML method instead
4. Check Confluence version (older versions may not support)

---

### ❌ Formatting looks wrong after HTML paste?

**Solutions:**
1. Click "Source" or "View source" in editor
2. Adjust table widths manually
3. Fix code block formatting
4. Use "Clear formatting" then reapply

---

### ❌ Draw.io plugin not installed?

**Solutions:**
1. Ask Confluence admin to install from Atlassian Marketplace
2. Search: "draw.io Diagrams for Confluence"
3. Alternative: Use app.diagrams.net → Export as PNG
4. Insert PNG image into Confluence

---

### ❌ Text diagrams look messy?

**Solutions:**
1. Use Code Block macro (not pre-formatted text)
2. Set language to "text" or "plain text"
3. Enable line numbers: NO
4. Enable word wrap: NO
5. Use fixed-width font
6. Adjust Confluence page width if too narrow

---

### ✅ For Confluence Storage Format (XML):
- Always copy the COMPLETE file

<h2 id="license">5. License Collection Mechanism</h2>

<div class="success-box">
    <h3>How License Data is Collected</h3>
    
    <h4>Step 1: Pod Discovery</h4>
    <ul>
        <li>License Service queries Kubernetes API to list all pods</li>
        <li>Filters pods that have license annotations</li>
        <li>Required annotations: productName, productID, productMetric, productChargedContainers</li>
    </ul>
    
    <h4>Step 2: Resource Calculation</h4>
    <ul>
        <li>Extracts CPU limits from charged containers in each pod</li>
        <li>Converts CPU limits to VPCs (Virtual Processor Cores)</li>
        <li>Formula: <strong>1000m (1 CPU core) = 1 VPC</strong></li>
        <li>Example: Container with 2000m CPU limit = 2 VPCs</li>
    </ul>
    
    <h4>Step 3: Aggregation</h4>
    <ul>
        <li>Groups usage by productName and productID</li>
        <li>Sums total VPCs per product across all pods</li>
        <li>Updates metrics every 30 minutes (polling interval)</li>
    </ul>
    
    <h4>Step 4: API Exposure</h4>
    <ul>
        <li>GET /status - Service health check</li>
        <li>GET /products - List of detected products</li>
        <li>POST /snapshot - Complete license usage snapshot (JSON)</li>
        <li>Authentication: Bearer token from Secret (ibm-licensing-token)</li>
    </ul>
</div>

<h2>5. License Collection Mechanism</h2>

<ac:structured-macro ac:name="panel">
  <ac:parameter ac:name="title">How License Data is Collected</ac:parameter>
  <ac:parameter ac:name="borderColor">#00875A</ac:parameter>
  <ac:rich-text-body>
    <p><strong>Step 1: Pod Discovery</strong></p>
    <ul>
      <li>License Service queries Kubernetes API to list all pods</li>
      <li>Filters pods that have license annotations</li>
      <li>Required annotations: productName, productID, productMetric, productChargedContainers</li>
    </ul>
    
    <p><strong>Step 2: Resource Calculation</strong></p>
    <ul>
      <li>Extracts CPU limits from charged containers in each pod</li>
      <li>Converts CPU limits to VPCs (Virtual Processor Cores)</li>
      <li>Formula: 1000m (1 CPU core) = 1 VPC</li>
      <li>Example: Container with 2000m CPU limit = 2 VPCs</li>
    </ul>
    
    <p><strong>Step 3: Aggregation</strong></p>
    <ul>
      <li>Groups usage by productName and productID</li>
      <li>Sums total VPCs per product across all pods</li>
      <li>Updates metrics every 30 minutes (polling interval)</li>
    </ul>
    
    <p><strong>Step 4: API Exposure</strong></p>
    <ul>
      <li>GET /status - Service health check</li>
      <li>GET /products - List of detected products</li>
      <li>POST /snapshot - Complete license usage snapshot (JSON)</li>
      <li>Authentication: Bearer token from Secret (ibm-licensing-token)</li>
    </ul>
  </ac:rich-text-body>
</ac:structured-macro>

<h2>6. Security Context Constraints (SCC)</h2>

<ac:structured-macro ac:name="panel">
  <ac:parameter ac:name="bgColor">#FFEBE6</ac:parameter>
  <ac:rich-text-body>
    <p><strong>Profile:</strong> restricted (OpenShift default SCC)</p>
    
    <p><strong>Allowed:</strong></p>
    <ul>
      <li>RunAsAny UID strategy</li>
      <li>Limited volume types: ConfigMap, Secret, EmptyDir</li>
      <li>Read-only root filesystem (where applicable)</li>
    </ul>
    
    <p><strong>Restricted (Not Allowed):</strong></p>
    <ul>
      <li>❌ No privileged containers</li>
      <li>❌ No host network access</li>
      <li>❌ No host PID/IPC access</li>
      <li>❌ No root user (unless explicitly configured)</li>
    </ul>
    
    <p><strong>Enforcement:</strong></p>
    <ul>
      <li>Applied via ServiceAccount isolation</li>
      <li>All operator and operand pods run under restricted SCC</li>
      <li>Validated by OpenShift admission controllers</li>
    </ul>

<h2>9. API Access</h2>

<ac:structured-macro ac:name="panel">
  <ac:parameter ac:name="title">Accessing the License Service API</ac:parameter>
  <ac:parameter ac:name="borderColor">#0052CC</ac:parameter>
  <ac:rich-text-body>
    <p><strong>Internal Access (within cluster):</strong></p>
    <ac:structured-macro ac:name="code">
      <ac:parameter ac:name="language">bash</ac:parameter>
      <ac:plain-text-body><![CDATA[
http://ibm-licensing-service-instance1.ibm-common-services.svc:8080
]]></ac:plain-text-body>
    </ac:structured-macro>
    
    <p><strong>External Access (via Route):</strong></p>
    <ac:structured-macro ac:name="code">
      <ac:parameter ac:name="language">bash</ac:parameter>
      <ac:plain-text-body><![CDATA[
# Get the route URL
oc get route ibm-licensing-service-instance1 -n ibm-common-services

# Get the API token
TOKEN=$(oc get secret ibm-licensing-token -n ibm-common-services -o jsonpath='{.data.token}' | base64 -d)

# Call the API
curl -k -H "Authorization: Bearer $TOKEN" \
  https://ibm-licensing-service-instance1-ibm-common-services.apps.cluster.domain/snapshot
]]></ac:plain-text-body>
    </ac:structured-macro>
    
    <p><strong>Available Endpoints:</strong></p>
    <ul>
      <li><code>GET /status</code> - Service health and version</li>
      <li><code>GET /products</code> - List of detected licensed products</li>
      <li><code>POST /snapshot</code> - Current license usage snapshot (JSON)</li>
    </ul>
  </ac:rich-text-body>
</ac:structured-macro>

]]></ac:plain-text-body>

# Check IBMLicensing CR

# Check operand pods
oc get pods -n ibm-common-services -l app.kubernetes.io/name=ibm-licensing

oc get svc,route -n ibm-common-services -l app.kubernetes.io/name=ibm-licensing

# Check operand logs
oc logs -n ibm-common-services -l app.kubernetes.io/name=ibm-licensing -c ibm-licensing

  --as=system:serviceaccount:ibm-common-services:ibm-licensing-service \

<h2>11. Important Notes</h2>

<ac:structured-macro ac:name="note">
  <ac:rich-text-body>
    <ul>
      <li><strong>Version:</strong> IBM License Operator v4.2.19 on OpenShift 4.16</li>
      <li><strong>Installation:</strong> Operator installed via OLM in ibm-common-services namespace (or custom)</li>
      <li><strong>Two Phases:</strong> Phase 1 deploys operator controller, Phase 2 deploys operand resources</li>
      <li><strong>RBAC:</strong> Operator has cluster-wide create/delete. Operand has cluster-wide READ only.</li>
      <li><strong>Security:</strong> All components run under OpenShift restricted SCC</li>
      <li><strong>Monitoring:</strong> License Service polls every 30 minutes for pod discovery</li>
      <li><strong>API:</strong> External access via Route with bearer token authentication</li>
    </ul>
  </ac:rich-text-body>
